# M2 Ports, UnitOfWork, User-Origin Flow Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** User can create project, persist intake chat, edit/lock foundation, manage character/fact/outline/reveal — all via ports + UnitOfWork + single write door (`commitCanonicalChangeSet`), tenant-scoped, no AI.

**Architecture:** `packages/application` owns port interfaces, UnitOfWork contract, use cases, `authorizeActiveUser`, progress reducer. `packages/db` implements ports with Prisma (tx-scoped), `dbNow`, and `createUnitOfWork`. Canon mutations only through `commitCanonicalChangeSet` (one PG tx): validate → apply ops → bump entity revisions → `project.currentCanonicalVersion += 1` once → AuditEvent + OutboxEvent. Server Actions in `apps/web` are thin adapters calling use cases.

**Tech Stack:** TypeScript 5.9 ESM, Prisma 7.9 + `@prisma/adapter-pg`, Vitest 4.1 + Testcontainers Postgres 16, pnpm 11, Next.js App Router Server Actions.

**Branch:** `feat/m2-ports-uow` (worktree `.worktrees/feat-m2-ports-uow`).

**Sources of truth:** D9 (UoW isolation/retry), S2.2 single write door, S4.1/S4.5, implementation-plan §6, verification-matrix rows: `fact-lifecycle`, `accept-proposal` (base +1), `outline-downstream`, `progress-view`, `idor`, `active-user-guard`, `concept-accept` (seeded).

```ts
  readonly code: AppErrorCode;
  readonly publicMessageCode: string;
  readonly httpStatus: number;
  readonly details?: Readonly<Record<string, unknown>>;
}

export function appError(
  code: AppErrorCode,
  publicMessageCode: string,
  httpStatus: number,
  details?: Readonly<Record<string, unknown>>,
): AppError {
  return details === undefined
    ? { code, publicMessageCode, httpStatus }
    : { code, publicMessageCode, httpStatus, details };
}

  return appError('NOT_FOUND', publicMessageCode, 404);
```

### commitCanonicalChangeSet (S2.2)

Input (application-level, already validated ops from core resolver or user-origin builder):

```ts
export interface CommitChangeSetInput {
  readonly projectId: string;
  readonly actorUserId: string;
  readonly origin: 'user' | 'ai' | 'system';
  readonly baseCanonicalVersion: number;
  readonly operationsHash: string;
  readonly operations: readonly CanonicalOpPersist[];
  readonly requestId: string;
}

export interface CanonicalOpPersist {
  readonly operationId: string;
  readonly ordinal: number;
  readonly operationType: string;
  readonly targetEntityType: string;
  readonly targetEntityId: string;
  readonly expectedRevision: number | null;
  readonly risk: string;
  readonly payload: unknown; // JSON-serializable canonical payload
}
```

Algorithm (one UoW transaction, serializable recommended for accept paths; read_committed + FOR UPDATE project row for M2 user-origin):

1. Lock project row `SELECT … FOR UPDATE` where `id = projectId AND owner_user_id = actorUserId AND deleted_at IS NULL`.
2. Assert `project.currentCanonicalVersion === baseCanonicalVersion` else `CAS_FAILED`.
3. Insert `CanonicalChangeSet` status=`pending` then apply each op in ordinal order (create/update tables + CAS `revision` where expectedRevision set).
4. Mark change set `status=applied`, `appliedCanonicalVersion = base + 1`.
5. `UPDATE projects SET current_canonical_version = current_canonical_version + 1, revision = revision + 1` **once**.
6. `audit.append({ action: 'change_set.applied', … })`.
7. `outbox.append({ eventType: 'canonical.change_set.applied', dedupeKey: \`changeset:${changeSetId}:applied\`, … })`.
8. Return `{ changeSetId, appliedCanonicalVersion }`.

Facts exist only after applied change set (`fact-lifecycle`).

```ts
// packages/application/src/authz/authorize-active-user.ts
import { err, ok, type Result } from '../result.js';
import { appError, type AppError } from '../errors.js';

export interface ActiveUser {
  readonly id: string;
  readonly status: 'active';
  readonly email: string;
}

export async function authorizeActiveUser(
  load: () => Promise<{ id: string; status: string; email: string } | null>,
): Promise<Result<ActiveUser, AppError>> {
  const user = await load();
  if (!user) {
    return err(appError('UNAUTHORIZED', 'msg.auth.unauthorized', 401));
  }
  if (user.status !== 'active') {
    return err(appError('FORBIDDEN', 'msg.auth.forbidden', 403, { status: user.status }));
  }
  return ok({ id: user.id, status: 'active', email: user.email });
}
```

```ts
import type { PrismaClient } from './client.js';

type DbClient = PrismaClient | Parameters<Parameters<PrismaClient['$transaction']>[0]>[0];

export async function dbNow(client: DbClient): Promise<Date> {
  const rows = await client.$queryRaw<Array<{ now: Date }>>`SELECT now() AS now`;
  const value = rows[0]?.now;
  if (!(value instanceof Date)) {
    throw new Error('dbNow: postgres now() did not return Date');
  }
  return value;
}
```

```ts
function isRetryableTxError(e: unknown): boolean {
  if (typeof e === 'object' && e !== null && 'code' in e) {
```
